사용자 여러분이 남겨주신 POC 개선의견이 반영된 항목에는 (POC #번호)를 표기했습니다.

  • 🎨 디자인검수 (명도대비 4.8:1) — 차시 상세에 🎨 디자인검수 버튼이 생겼습니다. 디자인 파일(이미지·PDF·PSD·AI)을 올리면 글자를 자동 인식해 글자↔배경 명도대비를 검사하고, 기준 4.8:1 미달 글자를 이미지 위 빨간 박스로 표시합니다. 이미지 클릭 → 핀 코멘트. (4장)

1시작하기 — 로그인과 화면 구성

CDMS는 웹 브라우저(크롬 권장)에서 사용합니다. 주소로 접속한 뒤 회사 이메일로 로그인합니다.

  1. 브라우저에서 cdms.mirimmedialab.co.kr 에 접속합니다.
  2. 회사 이메일과 비밀번호를 입력하고 로그인 을 누릅니다.
  3. 계정이 없으면 관리자(어드민)에게 초대를 요청하세요.

기본 화면 구성

  • 왼쪽 상단 “미림미디어랩 CDMS”를 누르면 언제든 첫 화면(전체 진행표)으로 돌아옵니다.
  • 보이는 메뉴는 권한(역할)에 따라 다릅니다. 담당자 초대·사용자 관리 등은 관리자에게만 보입니다.

2홈(전체 진행표) 보기

모든 사업과 그 안의 과정을 한눈에 봅니다. 과정을 고르면 상세 진행표로 들어갑니다.

  1. 왼쪽 ▤ 전체 진행표 (또는 상단 로고)를 누릅니다.
  2. 사업을 펼쳐 그 안의 과정(과목)을 확인합니다. 진행률과 품의완료 여부가 표시됩니다.
  3. 보려는 과정 이름을 클릭하면 그 과정의 진행표로 이동합니다.
상단 과목 드롭다운으로 과정을 빠르게 전환할 수 있고, ← 전체 진행표 로 목록으로 돌아옵니다.
🔍 검색 — 상단 검색창에 사업명·과정명·고객사·PM 이름을 입력하면 타이핑과 동시에 목록이 걸러집니다. 스크롤 없이 담당 사업을 바로 찾을 수 있습니다.

3과정별 진행표 읽는 법

한 과정의 모든 차시와 제작 단계 상태를 표로 봅니다. 칸의 색과 날짜로 진행 상황을 파악합니다.

칸(셀) 읽는 법

완료 — 초록 + 파일 생성 날짜 검수 — “검수 M/D” 수정 — 피드백 반영 필요 진행 — 작업 중 대기 — 아직 파일 없음(·)
  • 날짜 = NAS 파일이 만들어진 날입니다. 수정본이 생기면 아래에 “수정본 M/D”가 함께 표시됩니다.
  • 길이는 종편 영상의 재생 시간입니다(자동 인식).
  • 셀에 마우스를 올리면 어떤 NAS 파일이 매칭돼 표시된 건지(파일명·생성일)와, 파일을 지우면 어떻게 비우는지 안내가 툴팁으로 보입니다. 파일 매칭 없이 직접 체크한 셀은 “수동 표기”로 표시됩니다.
  • 상단 버튼: 📄 강의계획서 ✏ 과정,차시명 수정 👤 단계 담당자 🔄 NAS 동기화 🗂 NAS 폴더명 🔊 품질 점검
진행표는 10분마다 NAS와 자동 동기화됩니다(밤 12시~오전 6시 제외). 지금 바로 반영하려면 🔄 NAS 동기화 를 누르세요.

4차시 상세 — 제작 단계 확인·변경

진행표에서 차시(행)를 클릭하면 오른쪽에 상세 창이 열립니다. 각 제작 단계의 상태를 보고 바꿀 수 있습니다.

  1. 진행표에서 원하는 차시 줄을 클릭합니다.
  2. 각 단계 오른쪽의 상태 ▾ 에서 상태를 바꿉니다. “내 담당”으로 표시된 단계만 변경·저장됩니다.
  3. 단계 왼쪽 📁 를 누르면 그 단계의 파일을 보고 올릴 수 있습니다(다음 장).
  4. 맨 위 🎬 영상검수 로 영상 검수 화면을, 🎨 디자인검수 로 디자인 명도대비 검수 화면을 엽니다.
상태는 본인 담당 단계만 바꿀 수 있습니다. 담당이 아닌 단계는 읽기 전용으로 보입니다.
  • 완료일 수정 — 단계의 날짜(예: 6/12)를 클릭하면 완료 날짜를 직접 고칠 수 있습니다(담당자·관리자).
  • 종편 담당 지정 — 종편 단계의 ☐ 내 담당 버튼으로 그 차시 종편 담당자를 스스로 지정/해제할 수 있습니다. 담당자로 지정해 두면 그 차시에 검수 코멘트가 달릴 때마다 이메일 알림을 받습니다.
  • 🔊 품질 점검 — 상세 창 하단에 그 차시 종편 영상의 자동 품질 점검 결과(무음·클리핑·규격·명도대비 등)가 표시됩니다. 자세한 내용은 11장.
  • 🎨 디자인검수 (명도대비 ≥4.8:1) — 디자인 파일(이미지·PDF·PSD·AI)을 업로드하면 글자를 자동 인식(OCR)해 글자↔배경 명도대비를 검사합니다. 기준 4.8:1(영상 자막 기준 4.5:1보다 높음) 미달 글자는 이미지 위 빨간 박스로 표시되고, 최저 대비·통과(✓)/미달(✗) 판정이 요약됩니다. 이미지의 문제 지점을 클릭하면 번호 핀 코멘트를 남길 수 있고, 새 파일로 재검수하면 버전(v2, v3…)이 올라갑니다.

5단계별 파일 — 조회·다운로드·업로드

차시 상세의 단계 📁 를 누르면, 그 차시·단계의 NAS 파일을 보고 내려받거나 새로 올릴 수 있습니다.

파일 다운로드

  1. 단계 📁 를 눌러 패널을 엽니다(원하는 단계 탭 선택).
  2. 파일 오른쪽 ⬇ 를 누르면 내려받습니다 — 등록 당시의 원본 파일명·확장자 그대로 저장됩니다. (POC #27)
  3. 여러 파일 한 번에 — 파일 앞 체크박스를 여러 개 선택(Shift+클릭 = 범위 선택, 전체 선택 가능)하고 ⬇ 선택 다운로드 를 누르면 ZIP 파일 하나로 묶여 한 번에 받아집니다. (POC #33·#45·#65)

👁 미리보기 · 🖼 썸네일 (PSD·이미지)

  • PSD·이미지 파일 오른쪽 👁 를 누르면 다운로드 없이 미리보기가 열립니다(첫 생성 시 몇 초 소요, 이후엔 즉시).
  • 하단 🖼 썸네일 버튼은 현재 폴더의 PSD·이미지 전체를 격자 썸네일로 보여줍니다 — 원본 확인용으로 유용합니다.

📦 폴더 ZIP 다운로드 · 📁 폴더째 이동 · 🗑 선택 삭제

  • 폴더 앞 체크박스를 선택하고 ⬇ 선택 다운로드 를 누르면 폴더당 ZIP 1개로 압축해 다운로드할지, 파일을 하나씩 받을지 선택할 수 있습니다(하위 구조 유지). 수 GB급 초대형 폴더는 하위 폴더로 나눠 받는 것을 권장합니다. (POC #34·#53)
  • 📁→ 선택 이동 은 체크한 폴더 자체를 하위 구조 그대로 옮깁니다. 상위 폴더를 체크하면 하위는 자동 포함됩니다. (POC #51)
  • 🗑 선택 삭제 — 체크한 파일·폴더를 NAS의 99_휴지통 폴더로 이동합니다. 실제 삭제가 아니라 복원 가능하며, 사업 완료 14일 후 휴지통이 자동으로 비워집니다(NAS 쓰기 권한 필요). (POC #60)

파일 업로드

  1. 📤 파일 업로드 를 누르고 올릴 파일을 고릅니다 — 여러 개를 한 번에 선택하거나, 탐색기에서 파일들을 업로드 박스로 끌어다 놓아도 됩니다.
  2. 대용량 영상(25MB 초과)도 그대로 올릴 수 있습니다 — NAS로 직접 전송되며 진행률 %가 표시됩니다.
  3. 모든 업로드는 🛡 바이러스 검사(ClamAV)를 통과해야 작업 폴더에 반영됩니다. 감염 파일은 자동 격리되고 관리자에게 통보됩니다.
  4. 그 차시로 인식되지 않는 이름이면 앞에 “03차시_” 같은 태그가 자동으로 붙고, 종편 영상은 업로드 후 품질 점검이 자동 실행됩니다.
  5. 여러 파일 업로드가 끝나면 실패한 파일명과 사유 목록이 표시되고, 🛡 검사 완료/실패 결과가 올린 사람에게 메일로 전달됩니다. (POC #62)
안전장치 — 기존 파일은 절대 지워지거나 덮어쓰이지 않습니다. 같은 이름을 올리면 “이름 (2)”처럼 새 이름으로 저장됩니다.

목록 보기 개선

  • 파일 앞 아이콘이 종류별로 구분됩니다: 🎬 영상 · 📊 스토리보드/PPT · 🎨 디자인 원본 · 🎞️ 프리미어 프로젝트 · 🖼️ 이미지 · 🎵 오디오 · 💬 자막 · 📄 문서.
  • 하위 폴더(주차/차시/시안·분류·원본·old 등)가 📂 폴더별 그룹으로 구분 표시되고, 파일이 많으면 목록 안에서 스크롤됩니다.
  • 🕐 최신 수정순 정렬에서도 파일이 섞이지 않고 최상위 폴더(원본/최종 등)별로 묶여 표시됩니다 — 최근 수정된 파일을 폴더 맥락과 함께 확인할 수 있습니다. (POC #41·#61)
  • 🎞 영상소스 탭(과정 화면 버튼에서도 열림)에서 프리미어 프로젝트·효과음·에셋을 과정 공용으로 공유할 수 있습니다.

6영상 검수 (타임스탬프 코멘트)

영상을 보며 특정 시점·위치에 그림과 코멘트를 남기는 화면입니다. 검수 의견을 정확히 전달할 수 있습니다.

왼쪽 툴바 (위→아래)

  • ✎ 그리기 ON/OFF — 켜면 영상 위에 그릴 수 있습니다(켜면 영상은 일시정지).
  • ╱ 선 ▢ 박스 ➤ 화살표 📍 핀 — 표시 방법 선택.
  • 색상 — 기본은 빨강. 아래 색 동그라미로 바꿀 수 있습니다.

코멘트 남기기

  1. 차시 상세에서 🎬 영상검수 를 누릅니다(여러 클립이면 이어서 재생됩니다).
  2. 지적할 지점에서 영상을 일시정지합니다.
  3. 필요하면 툴바로 선·박스·화살표·핀을 그려 위치를 표시합니다.
  4. 오른쪽 입력창에 의견을 씁니다 — Enter = 줄바꿈이라 여러 수정 내용을 줄 바꿔 한 코멘트에 쓸 수 있습니다. ＋ 남기기 버튼(또는 Ctrl+Enter)으로 등록하면 그 시점(타임스탬프)에 저장됩니다. 답글도 동일합니다(등록 버튼).
  • 타임라인의 노란 점(●)이 코멘트가 있는 지점입니다. 점을 누르면 그 시점으로 이동합니다.
  • 검토할 때는 영상 위 표시 옆에 코멘트 내용이 함께 표시되어 어디를 지적했는지 바로 알 수 있습니다. 그림·말풍선은 그 코멘트의 시간 구간에서만 보이고, 구간을 벗어나면 자동으로 사라집니다.
  • 검수 상태는 진행중 · 검수요청 · 피드백필요(수정) · 완료 로 관리됩니다.
  • 코멘트를 남기면 그 차시의 종편 담당자에게 이메일로 전달됩니다(과정·차시·시점·내용 포함).

7버전 비교하기

같은 차시의 이전본과 수정본을 나란히 놓고 비교합니다. 무엇이 바뀌었는지 확인할 때 사용합니다.

  1. 영상 검수 화면에서 버전 비교하기 를 엽니다.
  2. 좌·우 각각에서 비교할 버전을 고릅니다(파일명 reN·수정 등을 자동 인식).
  3. 재생하며 달라진 부분을 확인합니다.

8NAS 폴더·동기화

CDMS는 NAS의 실제 파일·폴더와 연동됩니다. 폴더가 없는 새 과정은 단계 폴더를 만들어 연결합니다.

  1. 상단 ⚙ 단계·NAS 폴더 를 엽니다.
  2. ① 사용 단계를 켜고 끕니다(과정에 필요한 단계만).
  3. ② NAS 폴더 — 이미 종편 등 단계 폴더가 있으면 “이미 연결됨”으로 표시됩니다. 없으면 과정·단계 폴더 생성 으로 만듭니다.
  4. 진행표 상단 🔄 NAS 동기화 로 언제든 최신 상태를 불러옵니다.
  • 진행표는 10분마다 자동 동기화됩니다(밤 12시~오전 6시 제외).
  • 이미 단계 폴더가 있는 과정은 실수로 덮어쓰지 않도록 폴더 생성이 막혀 있습니다.
  • 삭제 반영 — NAS에서 파일을 지운 뒤 🔄 NAS 동기화 를 누르면 자동표기됐던 셀이 대기(빈 칸)로 초기화됩니다. 완료 알림에 “삭제 반영(비움) N건”이 표시됩니다. (자동 동기화는 안전을 위해 셀을 비우지 않습니다 — 반드시 수동 동기화로)

9단계 담당자·품의 알림 (관리자)

  • 단계 담당자 — 상단 👤 단계 담당자 에서 단계별 담당을 지정합니다. 품의 문서에서 자동 배치도 가능합니다.
  • 한 단계가 완료되면 그 차시의 다음 단계 담당자에게 자동 이메일이 갑니다.
  • 품의 지연 경고 메일 — PM이 지정됐지만 품의(전자결재)가 안 된 사업은 계약 시작 15일 경과 시 ⚠ 경고, 30일 경과 시 🚨 최종 경고 메일이 PM·관리자에게 각 1회 자동 발송됩니다.
  • 사용자·권한 — 관리자는 사용자 초대와 역할(PM·설계자·검수자·디자이너·외주 등)을 관리합니다.

10품질 점검 — 종편 영상 자동 검사

종편 영상을 CDMS로 업로드하면 바이러스 검사 통과 직후 자동으로 품질 점검이 실행되고, 문제가 있으면 업로드한 담당자에게 이메일이 갑니다.

검사 항목

  • 오디오 (v7 — 클립 단위 기준) — 3초 이상 무음 · 피크/클리핑(음 깨짐) · 오디오 트랙 없음 · 모노 출력·채널 무음(스테레오 한쪽) · 과대 음량 · 과소 음량 지속 · 클립 간 음량 밸런스(차시 안 클립끼리 크기 차이). ※ 이전의 ‘음량 급변’ 단일 기준은 클립 밸런스 기준으로 대체됐습니다.
  • 영상 규격 — 프레임 사이즈 1920×1080 · 비트전송률 3,000~5,500kbps · 프레임레이트 29.97~30fps 를 벗어나면 실제 값과 함께 경고
  • 블랙/화이트 프레임 — 100% 검은/흰 화면이 2프레임 이상 연속되면 구간 표시
  • 명도대비 — 자막·텍스트를 자동 인식해 글자/배경 대비가 4.5:1(WCAG AA) 미달인 구간 표시

실행 방법과 결과 확인

  1. 자동 — 종편 영상 업로드 시 그 차시가 자동 점검됩니다.
  2. 차시 단위 — 차시 상세 하단 🔊 이 차시 품질 점검 버튼.
  3. 과정 전체 — 과정 화면 상단 🔊 품질 점검 버튼(기본 NAS 과정 전용).
  4. 결과는 차시 상세의 🔊 품질 점검 섹션에 문제 유형·구간·상세로 표시되고, 구간 시간을 클릭하면 초반 재생 없이 곧바로 그 지점으로 이동한 뒤 재생됩니다. (POC #59)
  5. 문제 배지와 검수 타임라인 마커는 기준 유형별 색상으로 구분됩니다 — 빨강(무음·클리핑·오디오없음·모노/채널무음) · 주황(과대·과소음량·클립 밸런스) · 보라(규격) · 파랑(명도대비) · 회색(블랙/화이트 프레임).
  6. 명도대비 항목에는 문제 프레임 캡처(문제 영역 빨간 박스) 썸네일이 함께 표시됩니다 — 클릭하면 크게 보여 영상으로 이동하지 않고도 판단할 수 있습니다.
상세 기준·임계값은 저장소의 품질점검_기준.docx 문서를 참고하세요.

11POC 개선의견 남기기

사용 중 불편한 점·개선 아이디어를 화면 캡처와 함께 남기는 기능입니다. 좌측 하단 📝 POC 의견 버튼으로 엽니다. 외부(고객사·외주) 역할에게는 💬 CDMS 불편 사항 접수 버튼으로 표시됩니다.

  1. 의견을 입력합니다 — 굵게·글자색·목록 등 서식을 쓸 수 있습니다.
  2. 📷 화면 영역 캡처 를 누르면 POC 창이 내려가고 하단에 미니바가 뜹니다. 원하는 화면·팝업을 자유롭게 연 뒤 📷 지금 캡처 를 누르고 드래그로 영역을 선택하세요. 타 사이트 레퍼런스 등 별도 이미지는 🖼 이미지 첨부 로 올릴 수 있습니다. (POC #12·#20)
  3. 💾 의견 등록 — 매일 아침 9시 관리자에게 요약 메일이 갑니다.
  • 의견마다 #번호가 붙어 소통 기준이 되고, 각 의견에 💬 답변을 달 수 있습니다(작성자에게 메일 전달).
  • 상태는 접수 → ✔ 처리완료 → 🏁 최종 완료 로 관리되며, 최종 완료 건은 목록 아래로 내려갑니다.
  • 내용 수정·답변·상태 변경 모두 작성자와 관리자에게 이메일로 알립니다.
  • 외부 역할은 개인정보 보호를 위해 본인이 남긴 의견만 목록에 표시됩니다.
  • 지금까지 접수된 의견 60여 건 중 대부분이 반영 완료됐습니다 — 이번 업데이트의 파일 ZIP 다운로드·선택 삭제·정렬 그룹·업로드 실패 상세 등이 모두 POC 의견에서 나왔습니다.

12자주 묻는 질문(FAQ)

Q. 파일을 올렸는데 진행표에 바로 안 보여요.

10분 주기 동기화 후 반영됩니다. 급하면 🔄 NAS 동기화 를 누르세요.

Q. “이 과정에 NAS 폴더가 아직 없습니다”가 떠요.

그 과정은 아직 NAS 폴더가 연결되지 않은 것입니다. ⚙ 단계·NAS 폴더 에서 폴더를 만들거나(없을 때) 기존 폴더에 연결하세요.

Q. 상태 드롭다운이 안 바뀌어요.

본인 담당 단계만 변경할 수 있습니다. “내 담당” 표시가 없는 단계는 읽기 전용입니다.

Q. 같은 이름 파일을 올리면 기존 게 지워지나요?

아니요. 절대 지워지거나 덮어써지지 않고 새 이름으로 저장됩니다.

Q. 종편 열에 날짜가 안 떠요.

그 차시의 종편 영상이 NAS에 있어야 날짜가 표시됩니다. 동기화 후에도 안 뜨면 파일명/폴더를 확인하세요.

Q. 방금 올린 파일을 다운로드하면 깨져 있거나 열리지 않아요.

업로드 직후에는 🛡 바이러스 검사 대기 상태라 아직 작업 폴더에 반영되지 않았을 수 있습니다. 검사 완료 메일이 온 뒤(보통 수 분) 다시 받아보세요. 계속 깨져 있으면 관리자에게 알려주세요. (POC #64·#26)

Q. 여러 파일을 선택해 받았더니 ZIP 파일 하나로 와요.

정상 동작입니다. 브라우저가 연속 다운로드 중 마지막 1개만 저장하던 문제를 해결하기 위해 여러 파일은 ZIP 하나로 묶어 받도록 개선됐습니다(원본 파일명·폴더 구조 유지). 하나씩 받으려면 각 파일의 ⬇ 를 누르세요. (POC #65)
