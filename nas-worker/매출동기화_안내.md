# 매출 자동 동기화 (sales_sync.py) — 서버 cron, 완전 자동

'2026년도 매출 현황' 구글시트 → CDMS 사업(programs)으로 **고객명·계약금액·계약기간·PM명**을 매일 자동 반영합니다. 데스크톱 앱·승인과 무관하게 서버에서 24시간 돕니다.

## 1회 설정
1. **시트 게시(CSV)** — 시트 소유자가 1회:
   구글시트에서 **파일 → 공유 → 웹에 게시** → 시트 **'2026년'** 선택 → 형식 **CSV** → **게시** → 나오는 링크 복사.
   (예: `https://docs.google.com/spreadsheets/d/e/2PACX-.../pub?gid=545055052&single=true&output=csv`)
2. **서버 .env** (`/home/agent/.env`)에 추가:
   ```
   SALES_CSV_URL=<복사한 CSV 링크>
   ```
   (SUPABASE_URL / SUPABASE_SERVICE_ROLE_KEY 는 워커에 이미 있음)
3. **파일 배치**: `sales_sync.py` 를 `/home/agent/` 에 둔다(git pull 또는 scp).
4. **cron 등록** (매일 06:30):
   ```
   crontab -e
   30 6 * * *  cd /home/agent && /usr/bin/python3 sales_sync.py >> /home/agent/sales_sync.log 2>&1
   ```
5. **수동 테스트**: `cd /home/agent && python3 sales_sync.py` → "매출 동기화 완료: 갱신 N · 신규 M".

## 동작
- 매칭: 시트 **번호 = programs.seq** (year=2026).
- 기존 사업: client·amount·contract_start/end·pm_id(이름 매칭) 갱신. **사업명·과목·차시·진행률은 안 건드림.**
- 없는 번호: 신규 사업 생성(미등록).
- 표준 라이브러리만 사용(추가 설치 불필요).

## Cowork 예약작업과의 관계
- 같은 동기화를 하는 Cowork 예약작업(cdms-sales-sync)이 있다면, 이 서버 cron이 완전자동이므로 **Cowork 작업은 비활성화/삭제**해도 됩니다(둘 다 둬도 결과는 같음, 멱등).
